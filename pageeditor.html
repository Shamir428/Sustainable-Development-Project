My Role in the Project

In this group coursework, I was mainly responsible for building the more interactive and user-focused parts of the website. I focused on making sure that the website was not just informative, but also engaging for users. My main contribution was the Action Impact Simulator, along with creating the splash screen and the content page.

I tried to make my sections stand out by adding functionality and making sure the user actually interacts with the website rather than just reading information.

Pages I Developed

  • Splash Screen (index.html)
  • Action Impact Simulator (simulator.html)
  • Content Page (content.html)
  • Page Editor (this page)

All of these pages were designed to work together and give the website a clear structure. The splash screen acts as an introduction, the home page provides an overview, and the simulator and content page give more depth to the topic.

Splash Screen Implementation

For the splash screen, I wanted something simple but effective. I created a page that loads first and then automatically redirects the user to the main home page after a few seconds.

At first, I wasn’t fully sure how to do the automatic redirection, but I learned how to use a meta refresh tag to make it work. Once I understood that, it became straightforward.

Even though the splash screen is a small feature, I understand that it improves the overall user experience by giving a proper entry point into the website.

Action Impact Simulator

The Action Impact Simulator is the main feature I worked on and probably the most important part of my contribution. The idea behind it is to allow users to select different healthy actions and see how those choices affect their overall lifestyle score.

Each action is displayed as a card, and when the user clicks on a card, JavaScript is used to update the score. I made sure that each action could only be selected once, which required checking whether a card had already been clicked.

  • The score increases based on the selected actions
  • The same action cannot be selected multiple times
  • The score updates instantly on the screen
  • A message is displayed depending on the user’s total score

I also added a progress bar to make the simulator feel more interactive and visual. This shows the user how close they are to having a “healthy lifestyle” score.

Additionally, I implemented a reset button so users can restart the simulation and try different combinations of actions. I felt this was important because it allows users to experiment and better understand the impact of their choices.

Content Page Development

The content page was created to provide useful information about health and well-being. I made sure that the content was clear, structured, and easy to read.

Instead of just listing information, I tried to explain why health is important and how students are affected by poor habits such as lack of sleep, stress, and unhealthy eating.

I also linked the ideas in the content page to the simulator so that users can see how the information connects to the interactive feature. This makes the website feel more complete and meaningful.

Challenges and Solutions

One of the biggest challenges I faced was working with JavaScript, especially when trying to control how the cards behave when clicked. At the beginning, I had issues where users could click the same card multiple times, which would incorrectly increase the score.

To solve this, I used classList to check whether a card was already selected. This allowed me to stop duplicate clicks and made the simulator work properly.

Another challenge was improving the design. My first version looked very basic and did not look like a proper website. I improved this by using CSS features such as grid layouts, spacing, shadows, and hover effects to make it look more modern and professional.

Design Decisions

I chose a green colour theme because it is commonly associated with health, positivity, and growth. I wanted the design to match the theme of the website.

I also used card layouts throughout the site because they help organise content clearly and make it easier for users to understand information quickly.

Consistency was important to me, so I made sure that the simulator, content page, and home page all follow a similar design style.

Reflection

Overall, this project helped me improve both my technical skills and my understanding of how to build a complete website. I became more confident using JavaScript to create interactive features and learned how to structure pages properly using HTML and CSS.

I also improved my problem-solving skills by debugging issues and testing different solutions until everything worked correctly.

If I had more time, I would improve the design even further and possibly add more advanced features, but overall I am happy with what I have created.
