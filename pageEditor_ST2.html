Splash Screen

The splash screen was built using HTML elements such as div, headings, and paragraphs. CSS was used to create a full-screen background image and overlay for readability. This included using a background image with properties like background-size: cover and background-position: center to ensure it looked good on different screen sizes. An overlay was created using a semi-transparent div to improve text visibility against the background image. At first I was the mot worried about the splash screen as I had never built one before and I was not sure how to make it look good and also how to make it interactive, however after doing some research and looking at some examples, I was able to get a good understanding of how to build it and I am happy with the final result. I am happy with the final result and I think it is a great addition to our website as it provides a visually appealing introduction to our content and sets the tone for the rest of the site. The simulator was built using HTML structure with cards representing actions. CSS Grid was used to create a responsive layout and styling such as shadows and hover effects improved usability. The simulator was one of the funnest pages to build as it allowed me to be creative and also use JavaScript to make it interactive and engaging for users. I wanted to create a page that was not only informative but also fun and interactive, and I think the simulator achieves that goal. The use of cards to represent different actions was a great way to visually organize the content and make it easy for users to understand the impact of their choices. CSS Grid was particularly helpful in creating a responsive layout that looks good on different screen sizes, ensuring that users can access the simulator from any device. Overall, I am happy with how the simulator turned out and I believe it is a great addition to our website as it allows users to interact and see the impact of their actions in a fun and engaging way. Adding all the images and finding them was also fun making sure they are the correct size so they do not come as huge or too small was a bit of a challenge but I am happy with how they turned out and I think they add a great visual element to the simulator, making it more engaging for users.

JavaScript was used to handle interaction, update scores, prevent duplicate selections, and dynamically change the background image depending on the user’s impact level. This was a part of the project which required more time as it is a bit more challenging and I spent more time than I would have liked on it, however I am happy with how it turned out and I think it is a great addition to our website as it allows users to interact and see the impact of their actions in a fun and engaging way. The use of JavaScript to handle user interactions and update the simulator in real-time was a key aspect of making the simulator engaging and interactive. It allowed users to see the immediate impact of their choices, which adds an educational element to the simulator. Preventing duplicate selections was also important to ensure that users could not game the system and that the simulator provided an accurate representation of their impact level. Overall, I am pleased with how the JavaScript functionality turned out and I believe it enhances the overall user experience of the simulator.

Accessibility Considerations

Accessibility was considered by using alt text for images and semantic HTML elements. Colour contrast and layout ensure readability, and interactive elements are clearly visible. For example, all images used in the simulator and content pages include descriptive alt text to ensure that users with visual impairments can understand the content. Semantic HTML elements such as headings, sections, and lists were used to structure the content in a way that is easy to navigate for screen readers. Additionally, we ensured that there was sufficient colour contrast between text and background elements to enhance readability for users with visual impairments. Interactive elements such as buttons and links were styled to be clearly visible and distinguishable from non-interactive content, making it easier for users to interact with the website. Overall, we aimed to create an inclusive user experience by considering various accessibility factors throughout the design and development of our website.
