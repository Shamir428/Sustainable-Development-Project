Home Page

Link: View Home Page | View Validation

HTML Structure

The Home Page was built using semantic HTML elements including <header>, <main>, and <section> to clearly organise content. The layout is divided into three main sections: an introduction, a card-based content area, and a topic exploration section.

Each content block was grouped logically to improve readability and accessibility, allowing screen readers to navigate the page effectively.

CSS Techniques

CSS Grid was used to create a responsive card layout using grid-template-columns: repeat(auto-fit, minmax(...)), ensuring the layout adapts to different screen sizes.

Global CSS classes were used for reusable components such as cards, buttons, and navigation. Hover effects were implemented using transform and transition properties to improve interactivity.

Design and Implementation Decisions

A card-based layout was chosen to present key topics clearly and consistently. Each card includes an image, heading, and bullet points to break information into manageable sections.

Internal links were added to connect each card to its corresponding content page, improving navigation and user experience.

Gallery Page

Link: View Gallery | View Validation

HTML Structure

The Gallery Page uses a structured grid layout where each gallery item consists of an image and a caption. Each item is wrapped in a container to allow interaction through JavaScript.

The modal (extended view) is implemented as a hidden section that becomes visible when a user clicks on an image.

CSS Techniques

CSS Grid was used to create a responsive layout that automatically adjusts the number of columns depending on screen size.

Visual enhancements such as hover scaling effects were implemented using transform: scale(), and the modal overlay uses position: fixed with a semi-transparent background to focus user attention.

JavaScript Functionality

JavaScript functions were used to control the modal behaviour. When a user clicks a gallery item, the openModal() function dynamically updates the image source and text content.

Additional functions such as closeModal(), changeColor(), and changeFont() were implemented to enhance interactivity and allow basic user customisation.

Design Decisions

A modal design was chosen to allow users to view additional information without navigating away from the page. This improves usability and keeps the experience seamless.

Simple interactive controls were added to meet the specification requirement for user interaction and customisation.

Challenges and Solutions

One challenge was aligning the navigation bar correctly across all pages. This was resolved by using Flexbox and adjusting spacing within the header.

Another issue was the gallery initially appearing empty or poorly spaced. This was fixed by properly configuring the grid layout and ensuring images had correct sizing and margins.

I also faced difficulties maintaining consistent styling across pages, which was solved by refining the global CSS and reusing shared classes.

Accessibility Considerations

Accessibility was considered throughout development by using semantic HTML elements to improve structure and readability for assistive technologies.

All images include descriptive alt text to support users who rely on screen readers. Additionally, focus styles were implemented to allow keyboard navigation.

Colour contrast was also considered to ensure text is readable against background colours.

Compliance with JANET Regulations

All content included in the website is appropriate and relevant to the chosen SDG. External images were sourced from royalty-free platforms and are properly referenced.

The website avoids plagiarism by ensuring all written content is original. The design and implementation follow good web development practices and maintain a professional standard.
